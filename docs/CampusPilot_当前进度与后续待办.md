# 1\. 项目定位

项目名称：

```Plain Text
启航智伴 CampusPilot
```

项目方向：

```Plain Text
智慧校园
AI 原生
基于金蝶 AI 苍穹的新质校园智能体创新平台
```

我们要做的不是普通学生信息管理系统，而是一个面向高校学生成长与学业风险预警的智能体平台。

核心业务链路是：

```Plain Text
学生画像
→ 课程成绩
→ 学习行为
→ Agent 风险分析
→ 成长规划建议
→ 风险预警单
→ 辅导员/导师帮扶
→ 处理结果沉淀
→ 驾驶舱统计展示
```

当前第一版实现路线是：

```Plain Text
金蝶低代码业务对象承载数据
→ 知识库承载学生样例、课程成绩、学习行为和风险规则
→ Agent 根据知识库解释风险并生成建议
→ 风险预警单承载业务处理结果
```

后续要继续升级成：

```Plain Text
Agent 能查询业务对象
→ Agent 能自动判断风险
→ Agent 能生成结构化预警建议
→ 风险预警单能体现处理过程
→ 驾驶舱能展示风险分布和处理状态
→ 页面交互和视觉效果符合比赛演示要求
```

## 1\. Agent 与业务对象联动能力

### 目标

把当前“Agent \+ 知识库”的演示方式升级为“Agent 能查询金蝶业务对象并基于实时数据回答”。

当前 Agent 主要依赖知识库。

后续需要让 Agent 能读取学生画像、课程成绩、学习行为、风险预警单等业务对象数据。

### 需要实现的功能

第一阶段先实现查询类能力：

```Plain Text
query_student_profile：查询学生画像
query_course_score：查询课程成绩
query_learning_behavior：查询学习行为
query_warning_order：查询风险预警单
```

第二阶段再实现写入类能力：

```Plain Text
create_warning_order：创建风险预警单
update_warning_status：更新风险预警单状态
generate_growth_plan：生成学生成长规划
```

### 最小实现要求

如果时间有限，至少先实现：

```Plain Text
query_student_profile
query_warning_order
```

这样可以演示：

```Plain Text
用户问：张明远现在是什么情况？
→ Agent 查询学生画像
→ 返回 GPA、挂科数、出勤率、风险等级
→ Agent 生成结构化风险解释
```

### 更完整的实现效果

完整效果应该是：

```Plain Text
用户输入学生姓名或编号
→ Agent 查询学生画像
→ Agent 查询课程成绩
→ Agent 查询学习行为
→ Agent 根据规则判断风险等级
→ Agent 输出个性化帮扶建议
→ Agent 查询是否已有风险预警单
→ 如果没有，则建议创建预警单
```

### 操作指导

先确认这些业务对象的对象编码和字段标识：

```Plain Text
学生画像
课程成绩
学习行为
风险预警单
```

工具输入可以设计为：

```JSON
{
  "student_name": "张明远",
  "student_no": "STU2026001"
}
```

工具输出可以设计为：

```JSON
{
  "student_name": "张明远",
  "gpa": 2.31,
  "failed_count": 2,
  "attendance_rate": 0.76,
  "assignment_rate": 0.62,
  "risk_level": "高风险",
  "risk_reason": "高等数学和数据结构薄弱，存在专业基础断层风险"
}
```

Agent 提示词中需要加入：

```Plain Text
如果工具返回学生数据，必须优先基于工具结果回答。
如果工具无法查询，再基于知识库回答。
不要编造不存在的数据。
回答时要说明数据来源是工具查询还是知识库。
```

### 完成标准

完成后至少满足：

```Plain Text
1. Agent 能根据学生姓名或编号查询到学生数据
2. Agent 回答能引用查询结果
3. Agent 能根据查询结果判断风险
4. Agent 能说明是否建议生成风险预警单
```

## 2\. Agent 风险判断、成长规划与预警建议标准化

```Plain Text
AI建议
预警来源
处理状态
```

### 风险判断规则建议

可以统一成以下规则：

```Plain Text
高风险：
- GPA < 2.0
- 或挂科数 >= 2
- 或核心课程存在明显不及格
- 或出勤率 < 0.8 且作业完成率 < 0.7
- 或学分完成率明显低于年级正常进度

需要关注：
- GPA 处于 2.0 到 2.8
- 或出勤率低于 0.85
- 或作业完成率低于 0.8
- 或职业目标不明确
- 或科创参与度低
- 或学习行为近期下降

正常：
- GPA 稳定
- 无挂科
- 出勤和作业完成情况良好
- 有清晰成长方向
- 或曾有风险但近期改善明显
```

### Agent 回答格式建议

风险分析类问题固定输出：

```Plain Text
【风险等级】
【判断依据】
【风险原因】
【关键证据】
【帮扶建议】
【建议处理人】
【建议复评时间】
【可生成的风险预警单】
```

成长规划类问题固定输出：

```Plain Text

【主要短板】
【短期学习计划】
【中期成长计划】
【推荐课程】
【推荐竞赛或项目】
【需要老师支持的事项】
```

风险预警单建议固定输出：

```Plain Text
【预警单标题】
【学生编号】
【学生姓名】
【风险等级】
【风险分数】
【风险原因】
【AI建议】
【预警来源】
【初始处理状态】
```

### 操作指导

把这些规则同步到：


Agent 回答以下问题时格式稳定、内容具体：

