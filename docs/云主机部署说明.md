# CampusPilot 云主机部署说明

## 1. 上传目录

把这两个目录上传到云主机同一级目录，例如 `/opt/campuspilot/`：

```text
campuspilot-home/
campuspilot-server/
```

`campuspilot-home` 是前端静态页面，`campuspilot-server` 是 Java 后端。

## 2. 云主机需要的环境

- JDK 21 或以上。
- 允许访问后端端口，例如 `8787`。
- 如果要调用真实金蝶 Agent，需要云主机能访问金蝶 Agent API 地址。

## 3. Linux 运行示例

```bash
cd /opt/campuspilot/campuspilot-server
mkdir -p out
javac -encoding UTF-8 -d out $(find src/main/java -name "*.java")

export CAMPUSPILOT_HOST=0.0.0.0
export CAMPUSPILOT_PORT=8787
export CAMPUSPILOT_STATIC_ROOT=/opt/campuspilot/campuspilot-home
export CAMPUSPILOT_AGENT_API_URL="https://你的金蝶Agent接口地址"
export CAMPUSPILOT_AGENT_API_KEY="你的接口密钥"

java -cp out com.campuspilot.CampusPilotApplication
```

访问：

```text
http://云主机IP:8787/index.html#dashboard
```

## 4. 只改 Agent API 的部署方式

如果前端和 Java 后端代码不变，只需要改：

```text
CAMPUSPILOT_AGENT_API_URL
CAMPUSPILOT_AGENT_API_KEY
```

这两个变量就可以把本地兜底 Agent 切换成真实金蝶 Agent。

## 5. Nginx 可选反向代理

如果希望用 `80` 端口访问，可以让 Nginx 代理到 Java 后端：

```nginx
server {
    listen 80;
    server_name _;

    location / {
        proxy_pass http://127.0.0.1:8787;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
    }
}
```

## 6. 验证清单

- `GET /api/campuspilot/health` 返回 `ok=true`。
- 能打开首页和登录页。
- 登录后驾驶舱能加载学生、课程、行为、预警数据。
- 能生成预警单。
- 能完成确认、帮扶计划、学生反馈、复评结案。
- 配置 Agent API 后，问答接口能返回金蝶 Agent 的结果。

## 7. PostgreSQL 可选初始化

当前 Java 后端默认用内存数据，保证没有数据库也能演示。如果要接 PostgreSQL，可以先初始化表：

```bash
psql -U postgres -d campuspilot -f /opt/campuspilot/campuspilot-server/db/schema.sql
psql -U postgres -d campuspilot -f /opt/campuspilot/campuspilot-server/db/seed.sql
```

后续把 Java 仓储层从 `InMemoryCampusPilotStore` 替换为 JDBC 实现即可，接口路径和前端不需要变化。
